# Expected Output: Code Review Agent / CSV Invoice Export

A strong result should include:

- findings first, not a long overview
- clear Required vs Recommended separation
- file-level references tied back to spec or ADR concerns
- explicit security-surface flagging when relevant
- a route recommendation per finding type

Weak results:

- generic praise or summaries before findings
- style-only nitpicks treated as blockers
- review without reading the spec or ADR
