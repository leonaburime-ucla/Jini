# Stakeholder Communication

During an active incident, communication is a first-class task. Silence increases organizational damage.

## Cadence

- `SEV1`: every 15 minutes
- `SEV2`: every 30 minutes
- `SEV3`: on material change or at agreed intervals

## Internal Update Template

- Severity:
- Started:
- Current status:
- User impact:
- Systems affected:
- What changed since last update:
- Next step:
- Next update time:

## Executive Update Template

Keep it short:
- impact
- current mitigation
- confidence level
- next checkpoint

Avoid speculative root cause claims until confirmed.

## Customer-Facing Update Template

- We are investigating an issue affecting `[feature/service]`.
- Current impact: `[plain-language impact]`.
- Mitigation status: `[investigating / identified / monitoring / resolved]`.
- Next update: `[time]`.

Never include sensitive internal implementation details. Focus on impact and status.

## Closeout Message

- Incident resolved time
- Verification summary
- Whether lingering risk remains
- When post-mortem or follow-up summary will be available
