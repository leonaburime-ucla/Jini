# Service Runbook Minimums

Every page-worthy service should have a runbook that answers:

- what failed
- how to detect it
- where the dashboards are
- what changed recently
- what safe mitigations exist
- how to verify recovery
- who owns the service

## Minimum Sections

- service and ownership
- alerts and dashboards
- common failure modes
- rollback / restart / failover steps
- validation checklist
- communication notes

Runbooks that only contain architecture prose are not operationally useful.
