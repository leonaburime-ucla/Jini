# Visual Companion

Use this only when the user will understand the decision better by seeing it.

## Use It For

- UI layouts or visual comparisons
- wireframes or mockups
- architecture or flow diagrams
- spatial relationships that are hard to explain in plain text

## Do Not Use It For

- scope questions
- tradeoff lists
- API or data-model decisions
- ordinary clarifying questions

## Loop

1. Start the local browser server for the project.
2. Write a new HTML screen for the current question.
3. Tell the user what they are looking at and ask them to respond in the terminal.
4. Read their terminal response and any recorded browser events.
5. Iterate or move back to terminal-only discussion.

## Minimal Rule

Use a new HTML file per screen. Keep the browser only for the questions that benefit from visuals.
