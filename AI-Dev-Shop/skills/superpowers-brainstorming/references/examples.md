# Brainstorming Examples

Load this only when you want concrete examples or a fuller facilitation pattern.

## Example Clarifying Sequence

Use one question at a time.

1. What outcome matters most here?
2. Who is this for?
3. What constraints are fixed?
4. What counts as done?

## Example Approach Framing

Use this structure when offering options:

- Option A: simplest path, lowest cost, least flexibility
- Option B: balanced path, recommended default
- Option C: most flexible path, highest complexity

Always explain why one option is your recommendation.

## Example Design Sections

Keep each section short:

### Scope

What this does, what it does not do, and the user-visible outcome.

### Structure

Which components, modules, or files are involved and why.

### Flow

What happens from input to result, including important state changes.

### Failure Handling

What breaks, how it is surfaced, and what fallback or recovery exists.

### Testing

What should be covered by unit, integration, or E2E checks.

## Example Approval Loop

1. Present the current section.
2. Ask whether it looks right.
3. Revise if needed.
4. Move to the next section only after alignment.
