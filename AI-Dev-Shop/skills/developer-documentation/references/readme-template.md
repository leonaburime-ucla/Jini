# README Template

Use this for the primary entry doc of a repository, package, or service.

## Recommended Structure

1. Title
2. One-sentence value statement
3. Quickstart
4. Installation / prerequisites
5. Basic usage
6. Configuration
7. Advanced usage or architecture notes
8. Link to full API/reference docs
9. Contributing / development notes

## 5-Second Test

Within the first screen a reader should be able to answer:
- What is this?
- Why would I use it?
- How do I start?

## Quickstart Rules

- Use the shortest viable path
- Include commands exactly as they should be run
- Show the expected visible outcome
- Defer nuance and edge cases until after the reader gets a working result

## Anti-Patterns

- Long marketing intro before install
- Configuration details before first success
- Example commands that do not reflect the current codebase
- Hiding required environment variables until late in the doc
