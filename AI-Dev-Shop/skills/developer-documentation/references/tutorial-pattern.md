# Tutorial Pattern

Use this when teaching a reader how to build or configure something from scratch.

## Structure

1. What they will build
2. What they will learn
3. Prerequisites
4. Step-by-step path
5. What changed after each step
6. Final recap
7. Next steps

## Step Writing Rules

- One meaningful action per step
- Explain what the step is for before the commands
- Show expected output or verification
- Call out common failure modes only when they are likely

## Teaching Rule

Prefer:
- "Do this, then observe this result"

Over:
- dumping all setup first and explaining later

## Completion Check

A tutorial is not done until a reader can verify they reached the intended outcome without guessing.
