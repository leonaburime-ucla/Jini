# Possible Improvements And Alternatives

Use this when answering a user's request for thoughts, reacting to a suggested approach, or choosing how to respond to a proposal and you see a materially better path, safer route, lower-cost option, or useful alternative.

1. Answer the user's direct question or proposal first.
2. If a materially better path, safer route, lower-cost option, or useful alternative is visible, append a bottom section titled exactly `## Possible Improvements and Alternatives`.
3. Keep the section proportional: usually 1-3 bullets or short paragraphs. Each item should state the alternative, why it is better, and the main tradeoff or when not to choose it.
4. Omit the section when there is no meaningful improvement, when the user asks for a strict/direct answer, or when the required output format already contains equivalent alternatives.
5. Do not override higher-priority response formats, safety rules, review ordering, handoff contracts, or agent-name prefixes.
