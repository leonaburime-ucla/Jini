# Design Shotgun

## When

Use when the user wants visual options, competing design directions, or a fast comparison before committing to one direction.

## Workflow

1. Confirm the target surface and constraints: product, audience, platform, existing UI, and non-negotiable brand or accessibility rules.
2. Generate 3-6 meaningfully distinct directions. Avoid small variations of the same idea.
3. For each direction, provide:
   - name
   - core idea
   - visual language
   - interaction posture
   - accessibility and implementation tradeoffs
4. Compare directions across brand fit, clarity, speed to build, accessibility, novelty, and maintenance cost.
5. Ask the user for structured preference feedback: keep, reject, combine, or revise.
6. Capture taste signals as explicit notes so the next mode can reuse them.

## Output

- Variant list
- Comparison matrix
- Recommended short list
- Preference questions
- Next step recommendation

## Guardrails

- Do not auto-select a final direction without user feedback.
- Do not generate implementation code in this mode.
- Do not present impossible or inaccessible designs as production-ready.
