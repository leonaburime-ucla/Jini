# Product Validation in Prototype Mode

Use this when the prototype exists to test a hypothesis, not just to show a UI.

## Focus

Prototype the smallest thing that can answer:
- Do users understand the value?
- Can they complete the core flow?
- What signal determines whether this idea is worth hardening?

## Add These Early

- one feedback collection point
- one meaningful usage event
- one clear success metric

## Avoid

- polishing low-value edges before the core flow works
- building infrastructure for scenarios the prototype is not testing
- treating prototype usage as proof of production readiness
