# Design Consultation

## When

Use for a new product surface, design direction, brand/aesthetic pass, or design-system proposal before implementation.

## Workflow

1. Gather product context: target users, core workflow, business goal, platform, existing brand, and UI constraints.
2. Inspect existing UI or design artifacts when available before proposing a new direction.
3. Identify the design problem in plain terms: what must be easier, clearer, faster, more trusted, or more memorable.
4. Optionally research comparable products or patterns when the user asks for market/landscape awareness.
5. Propose one primary design direction and one credible alternative.
6. Define foundations only as far as needed: layout principles, typography direction, color role, spacing scale, motion posture, and component tone.
7. Call out tradeoffs: implementation complexity, accessibility risk, brand risk, and where taste decisions need human input.

## Output

- Context summary
- Recommended design direction
- Alternative direction
- Foundations and component implications
- Decisions needed from the user
- Suggested next mode: `shotgun`, `html`, or `review`

## Guardrails

- Do not invent user research or brand evidence.
- Do not prescribe a full redesign when a smaller design repair would solve the problem.
- Do not pick decorative visual trends over usability, accessibility, or product fit.
