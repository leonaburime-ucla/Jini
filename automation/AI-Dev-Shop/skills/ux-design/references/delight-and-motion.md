# Delight and Motion

Use this reference when the product should feel lively, memorable, or distinctive without sacrificing usability.

## Default Rule

Delight is optional. Clarity, speed, and accessibility are not.

## Appropriate Uses

- celebratory success states
- onboarding encouragement
- empty states that benefit from warmth
- lightweight hover/focus feedback
- brand personality in low-risk surfaces

## Motion Rules

Motion should:

- clarify state change
- reinforce hierarchy
- acknowledge user action
- reduce abruptness

Motion should not:

- block task completion
- hide information
- trigger distraction loops
- ignore reduced-motion preferences

## Microcopy Rules

Use personality carefully in:

- loading text
- empty states
- success confirmations
- low-severity errors

Keep copy:

- short
- helpful
- brand-consistent
- non-patronizing

Avoid jokey or cute copy in high-stress flows such as payments, security, legal consent, or destructive actions.

## Delight Spectrum

- subtle: hover, focus, small success feedback
- moderate: celebratory completion feedback, richer empty states, guided encouragement
- strong: campaign-specific personality, easter eggs, playful brand moments

Default to subtle unless the product context supports more.

## Anti-Patterns

- confetti for routine actions
- whimsical error text during serious failure states
- animations that slow repeated workflows
- easter eggs on business-critical surfaces
- personality that breaks brand trust
