# Research and Validation

Use this reference when the design needs a validation plan, clearer user framing, or a way to separate assumptions from evidence.

## Important Constraint

Do not present invented research as completed research. Use this reference to structure:

- proposed research plans
- user hypotheses
- validation criteria
- usability-test plans

## Useful Artifacts

### Proto-Persona

Use when real research does not yet exist.

Include:

- user type
- main goals
- main frustrations
- environment/context of use
- skill level
- critical accessibility considerations

Label it explicitly as a working assumption.

### Journey Snapshot

Map:

- trigger
- entry point
- main steps
- friction points
- moments of uncertainty
- completion state
- follow-up state

### Usability-Test Plan

Define:

- research questions
- participant criteria
- tasks to complete
- success criteria
- metrics to watch
- qualitative prompts

### Validation Checklist

Use before handoff:

- does the layout support the primary user goal quickly
- are key decisions reversible
- do labels match user language
- are errors recoverable
- is the empty state actionable
- is keyboard/mobile usage still workable

## When This Reference Is Worth Loading

- dashboard or workflow-heavy UI
- ambiguous user roles
- onboarding flows
- high-risk UX with multiple steps
- claims about "user needs" that need structure

## Anti-Patterns

- pretending empathy maps are evidence
- writing personas with no behavioral relevance
- validation plans with no measurable success criteria
- feature flows optimized for internal assumptions only
